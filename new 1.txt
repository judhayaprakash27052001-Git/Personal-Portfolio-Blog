<!-- Steps -->
<section class="container" id="steps">
  <h2 class="mb-5" data-aos="fade-up">Step-by-Step with Screenshots</h2>

  <!-- Step 1 -->
  <div class="step" data-aos="fade-up" data-aos-delay="100">
    <div class="step-title"><i class="bi bi-ui-checks"></i> Step 1: Design the Contact Form</div>
    <ul>
      <li>Create an HTML page with a form containing fields for name, email, and message and include "Save" and "View Messages" controls.</li>
<li>Add JavaScript that sends a POST request to save contact details and a GET request to retrieve messages.</li>
<li>Place a screenshot or wireframe showing the form and UI interaction.</li>
      <li>Include Name, Email, and Message fields.</li>
      <li>Use Bootstrap for responsive design.</li>
      <li>Add form validation and accessibility labels.</li>
    </ul>
    <img src="assets/img/fontend.png" alt="Contact Form UI Screenshot">
  </div>

  <!-- Step 2 -->
  <div class="step" data-aos="fade-up" data-aos-delay="200">
    <div class="step-title"><i class="bi bi-diagram-3"></i> Step 2: Create DynamoDB Table</div>
    <ul>
      <li>Open the AWS Console → DynamoDB → Create table.</li>
      <li>Table name: Serverless</li>
      <li>Primary key: msg (String).</li>
      <li>Use on-demand capacity mode for simplicity.</li>
    </ul>
    <img src="assets/img/DynamoDB-Table.png" alt="API Gateway Setup Screenshot">
  </div>

  <!-- Step 3 -->
  <div class="step" data-aos="fade-up" data-aos-delay="300">
    <div class="step-title"><i class="bi bi-lambda"></i> Step 3: Write Lambda Function</div>
      <li>Log in to AWS Console & open API Gateway.</li>
      <li>Create a new REST API while creating lambda post and get attach role with permissions eg : DynamoDB full access for lambda to read and write.</li>
      <li>POST Lambda — receives form data and stores it in DynamoDB with a unique ID .</li>
	  <li>GET Lambda — retrieves saved contact entries from DynamoDB.</li>
      <li>Enable CORS for frontend access and assign the Lambda execution role the necessary DynamoDB permissions and test using sample payloads.</li>
      <li>Deploy API to a stage (e.g, prod).</li>
	  
	  GET Lambda IMAGE
	  
    <img src="assets/img/GETLAMBDA.png" alt="Lambda Function Screenshot">
	
	  POST Lambda IMAGE
	 
	<img src="assets/img/POSTLAMBDA.png" alt="Lambda Function Screenshot">
  </div>

  <!-- Step 4 -->
  <div class="step" data-aos="fade-up" data-aos-delay="400">
    <div class="step-title"><i class="bi bi-database-check"></i> Step 4: Set Up API Gateway</div>
    <ul>
       <li>Log in to AWS Console & open API Gateway.</li>
      <li>Create a new REST API.</li>
	  <li>Create an HTTP API in API Gateway.</li>
      <li>Create two routes:POST /contacts → POST Lambda.</li>
      <li>GET /contacts → GET Lambda.</li>
      <li>Enable CORS for your frontend origin (or * for testing), deploy the API, and copy the invoke URL(s)and past it in JavaScript api url point.</li>
	  
    </ul>
    <img src="assets/img/apigateway.png" alt="DynamoDB Table Screenshot">
  </div>

  <!-- Step 5 -->
  <div class="step" data-aos="fade-up" data-aos-delay="500">
    <div class="step-title"><i class="bi bi-bug-fill"></i> Step 5: Deploy Frontend to S3</div>
    <ul>
      <li>Create an S3 bucket (globally unique name).</li>
      <li>Enable static website hosting in the bucket properties.</li>
      <li>Configure caching, HTTPS, and (optionally) an origin access identity.</li>
	  <li>Use the CloudFront domain name as the public URL for the app (or map a custom domain).</li>
    </ul>
    <img src="assets/img/S3BUCKET.png" alt="Testing Screenshot">
  </div>

  <!-- Step 6 -->
  <div class="step" data-aos="fade-up" data-aos-delay="600">
    <div class="step-title"><i class="bi bi-cloud-upload"></i> Step 6: Deploy Frontend to S3 + CloudFront</div>
    <ul>
      <li>Create a CloudFront distribution using the S3 website or S3 origin.</li>
      <li>Set Default Root Object to index.html.</li>
      <li>Enable static website hosting.</li>
      <li>Create a CloudFront distribution pointing to the S3 bucket.</li>
      <li>Use CloudFront URL as your project domain.</li>
    </ul>
    <img src="assets/img/Cloudfont-S3-Link.png" alt="S3 and CloudFront Deployment Screenshot">
  </div>

  <!-- Step 7 -->
  <div class="step" data-aos="fade-up" data-aos-delay="700">
    <div class="step-title"><i class="bi bi-shield-lock"></i> Step 7: (Optional) Configure a Custom Domain with Route 53</div>
    <ul>
      <li>Register or use an existing domain in Route 53.</li>
      <li>Create an Alias record pointing to the CloudFront distribution.</li>
      <li>Request/attach an ACM certificate (us-east-1 for CloudFront) and enable HTTPS.</li>
      <li>Monitor DynamoDB usage for CloudWatch.</li>
    </ul>
    <img src="assets/img/Route53.png" alt="Security and Monitoring Screenshot">
  </div>
